## 2. Implementation Summary

[src/engine/UreVM.ts](../../../src/engine/UreVM.ts) is a pure-TypeScript simulator (no FPGA), structured in five sections:

- **Quaternion primitives** (lines 12-60): `Quaternion` interface `{w, x, y, z}`, plus `quatAdd`, `quatSub`, `quatMul` (full Hamilton product), `quatScale`, `quatNorm`, `quatNormalize`, `quatConjugate`, `quatDot`.
- **Predicate planes** (lines 65-66): `PredicatePlane` type `'RR' | 'RI' | 'IR' | 'II'`, cycled by opcode `0x1E PLANE_ROTATE`.
- **Pendinium primes** (lines 71-75): 30-prime array, all `p ≡ 1 (mod 12)`, range [13, 547].
- **VM state** (lines 80-107): four quaternion registers (`alpha`/`beta`/`gamma`/`delta`), `accumulator`, `nullLedger`, `tick`, `cycle`, `halted`, `forbidden361`, `plane`, `foldAngle`, `parityBit`, `log[]`, `pc`.
- **Opcode table** (lines 139-910): 72 opcodes indexed `0x00` through `0x47`. See section 3 below for spec-to-opcode mapping.

Runtime: `createVM()` seeds registers to basis quaternions. `executeOpcode(state, idx)` applies the op, advances tick mod 370, detects State 361 (resets parity), updates null ledger, appends to rolling 100-entry log, advances PC. `runVM(state, ticks)` drives execution. `seedFromBlip(state, signal, coherence, entropy, qWeight)` is the integration point from radar telemetry. `vmSnapshot(state)` produces a UI-ready summary with ledger-balance classification (BALANCED < 0.01 < NOMINAL < 0.5 < DRIFT < 2 < CRITICAL).
