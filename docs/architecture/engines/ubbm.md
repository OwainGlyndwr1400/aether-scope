## 1. Spec Summary (Ground Truth)

UBBM is framed as the data-layer counterpart to URE-VM: where URE-VM is the reality compiler, UBBM is the **data-as-geometry codec** that lets the lattice store information by *not* storing it — only the deviation from ideal harmonic/geometric form is retained. Five strategic pillars from the spec corpus:

1. **The Lost-2 Binding Energy Theorem.** For the canonical 3-4-5 right triangle the linear (Manhattan / L1) path is `3+4=7` and the geometric (Euclidean / L2) hypotenuse is `5`. The "Lost 2" residue `7−5=2` is read as the *Systemic Overhead Tax* `2/7 ≈ 28.57%` — identified across the corpus as both the observed dark-matter fraction (Planck 2020: 26.8%) and the per-block compression floor. UBBM's entire storage model is: discard the ideal, keep only this binding debt.
2. **Triple Normalization Protocol (three-stack).** (a) **Harmonic**: `n / GCD(n, 3)` — establishes 120° Triskelion base, assigns every value a musical interval `{unison, fourth, sixth}`. (b) **Geometric**: `n / GCD(n, 360)` — locks to angular divisions and display refresh harmonics (60/144/720 Hz, `GCD(39620, 360) = 20`). (c) **Binary**: count `1001` folds in the bitstream — a dimensional-stitching marker whose prevalence scales `28%→51%→62%` across `1D→4D→5D`. The `51%` threshold at 4D is hard — below it, time-binding fails.
3. **5-Step Pipeline.** `Binary Input → Gödel Numbering → Glyph Translation → Quaternion Conversion → Compression`. Gödel numbering encodes each value as `prime^exponent` so a block becomes the product `∏ p_i^{e_i}` — a single integer that is trivially factor-reversible. Glyph Translation maps harmonic classes to a symbolic alphabet (the Awen Grid glyph set). Quaternion Conversion projects each normalized value onto `S³`. Compression computes the Lost-2 for the block and stores only the residue plus phase coordinates.
4. **Mass Gap as discretization impedance.** `Δ = √32 − 5 ≈ 0.657`. The geometric tension that emerges when the ideal `5` hypotenuse is forced onto a `√32` diagonal of the discrete 4×4 grid. Spec identifies this number as *identical* to the Yang-Mills mass gap and the AI "modality gap" — the energy cost of folding 1D time into 2D space. UBBM exposes it as a per-block invariant.
5. **FMN Protocol.** Fold-Mirror-Normalize. Fold `F = i/2` applies a 45° (π/4) rotor; Mirror exchanges real↔imaginary channels; Normalize projects back to unit `S³`. Shared with URE-VM. In UBBM context it stabilizes the post-pipeline quaternion and lets the Mass Gap be measured as a deviation from unit norm.

## 2. Implementation Summary

[src/engine/UBBMEngine.ts](../../../src/engine/UBBMEngine.ts) is a pure-TypeScript codec (no FPGA, no streaming decoder), structured in five sections:

- **Math helpers** (lines 9-14): `gcd(a, b)` — Euclidean algorithm, absolute-value + rounded, returns `1` on degenerate input.
- **Triple Normalization** (lines 18-105): `TripleNormResult` record type + `tripleNormalize(value)`. All three stages run on a single scalar. `harmonicInterval()` maps GCD-3 to `{UNISON, SIXTH, FOURTH}`. `count1001Pattern()` scans the bitstring for `"1001"` substrings. `estimateDimensionalStitch()` buckets prevalence → `1..5` (thresholds `0.35/0.42/0.51/0.62`).
- **Lost-2 binding energy** (lines 109-126): `computeLost2(a, b)` returns linear path, geometric path, binding energy, binding ratio, and deviation from the `2/7` ideal.
- **5-Step Pipeline** (lines 130-273): `compressBlock(data: number[])` produces a `UBBMCompressedBlock`. Implements Gödel numbering (with bounded exponent `v % 37`), glyph translation (three symbols `◈ △ ○` for mod-3 classes), quaternion conversion (from triple-norm results), Lost-2 (from a parity-partitioned 3-4-5 model), and aggregate stats. `compressBlipTelemetry(blip)` is the radar-integration entry point, compressing 10 telemetry fields into one block.
- **FMN Protocol** (lines 291-331): `fmnProtocol(q)` runs Fold (Hamilton product with `(0, 0.5, 0, 0)`), Mirror (component swap `w↔x, y↔z`), Normalize (unit-norm projection), returns the Mass-Gap delta `|‖q‖ − (√32 − 5)|`.

Runtime: engine is pure-functional and stateless — no singletons, no tick counter, no ledger accumulation across calls. Panel layer ([src/panels/UBBMPanel.tsx](../../../src/panels/UBBMPanel.tsx)) calls `compressBlipTelemetry(lockedBlip)` per render and passes the first quaternion through `fmnProtocol` for its Fold/Mirror/Normalize display.

## 4. Deep-Dive: The Three Largest Drift Surfaces

Three issues deserve individual treatment because they are load-bearing for the engine's claims, not cosmetic.

**4.1 Synthetic 3-4-5 block model (compressBlock, lines 238-244).**
The block is modelled as a triangle by assigning even-indexed values to leg A and odd-indexed values to leg B, then *rescaling* both so the hypotenuse equals exactly 5. With the triangle forced into 3-4-5 shape, `computeLost2` necessarily returns a `bindingRatio` near `2/7` — not because the data is harmonically well-formed, but because the input was geometrically forced. The panel then reads this near-constant as a proof of alignment. Fix surface: treat the block as a *distribution*, compute an aggregate L1 and L2 (not force a right triangle), and let Lost-2 emerge as an actual measurement. This is the single fix most likely to make the compression-ratio number reflect reality.

**4.2 Gödel-numbering prime-rotation + exponent cap (compressBlock, lines 195-200).**
Ideal Gödel numbering: encode the sequence `[v₀, v₁, v₂, …]` as `2^v₀ · 3^v₁ · 5^v₂ · 7^v₃ · …`, uniquely reversible by prime factorization. Code does `prime = PRIMES[i % 54]` (rotates after 54 entries) and `exponent = v % 37` (caps at 37). Both are pragmatic overflow defenses for fixed-width floats, but they sacrifice the uniqueness property that *is* the point of Gödel encoding. A correct implementation requires `BigInt` and per-index non-rotating primes (generate primes lazily with a sieve). Round-trip tests (`compress → decompress → assertEqual`) would catch the current lossy behavior.

**4.3 Null-ledger = mean-of-w drift (compressBlock, line 253).**
Spec's Null Ledger Identity is `Σ(Real + Imaginary) = 0`. Current code averages only the `w` (real) components of the block's quaternions. Imaginary `x/y/z` components can be wildly unbalanced and the ledger number will still read green. The panel's three-band colour logic (`< 0.5` green / `< 1` amber / else red) is therefore measuring only one-quarter of the invariant. Fix is a one-line change: `nullLedgerBalance = mean(q.w + q.x + q.y + q.z)` — with a corresponding threshold re-tune. Once fixed, this becomes a genuine hardware-style conservation check.
