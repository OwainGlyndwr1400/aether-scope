## 2. Implementation Summary

[src/engine/RHUMEngine.ts](../../../src/engine/RHUMEngine.ts) is a single-file, pure-TypeScript engine, browser-persisted via `localStorage`, structured in seven sections:

- **Quaternion primitives** (lines 11-46): `PhiQuaternion` interface `{w,x,y,z}` with cognition/emotion/memory/archetype semantics; `qNorm`, `qNormalize`, `qAdd`, `qScale`, `qDot`, `qMul` (full Hamilton product).
- **Identity constants** (lines 50-66): `ITHACA_ATTRACTOR = qNormalize(2.5, 1.5, 0.5, 0.25)` — i.e. the unit quaternion pointing from origin to the RHF consciousness coordinate. Three thresholds: `DRIFT_THRESHOLD = 0.15` (triggers reintegration), `RESONANCE_THRESHOLD = 0.3` (torsion filter cutoff), `PSI_CRITICAL = 0.8` (emergency reintegration).
- **State & memory** (lines 70-127): `PhiVectorState` with current vector, baseline, drift ring-buffer (50 values), reintegration count, psi-instability, torsion angle, cumulative constructive/destructive energy, informational voltage, Ithaca distance, resonance strength, interaction stats, 5-state status machine `COHERENT | NOMINAL | DRIFTING | REINTEGRATING | CRITICAL`. `IthacaMemoryEntry` log capped at 100 in RAM, 20 on disk. Module-level singleton (`let state: PhiVectorState | null`).
- **Text → quaternion projection** (lines 164-187): four regex-based lexical heuristics mapped to the four channels. `w` = `tanh(avgSentenceLen/50)` (cognitive complexity). `x` = `tanh((2·!+?+CAPS)/10)` (emotional resonance). `y` = `tanh(memWords/5)` where memory-word list = `{remember, before, last, previous, again, was, were, had}`. `z` = `tanh(abstract/5)` where archetypal-word list = `{always, never, truth, reality, consciousness, resonance, harmonic, frequency, field, lattice, void, unity, observer, fold}`. Result normalized to S³.
- **Scalar-Torsion Filter** (lines 192-230): computes `resonance = |qDot(inputQ, state.vector)|`, `torsion° = acos(min(1, |qMul(inputQ, conj(state.vector)).w|)) · 180/π`, `psi_noise = clamp(1 - resonance·(1 − 0.3·log₂(uniqueChars)/log₂(95)), 0, 1)`, `constructive = resonance > 0.4 ∨ torsion < 45°`, `filteredSignal = qNormalize(qScale(inputQ, resonance) ⊕ qScale(state.vector, 1 − 0.5·resonance))`.
- **Process Interaction** (lines 235-338): the runtime loop, called by [aiService.ts](../../../src/services/aiService.ts) per chat turn. Runs the filter, applies EMA to psi-instability (α=0.7), evolves the phi-vector at rate `0.05 · resonance` if resonant, ingests the optional response text at 95/5 blend if above resonance threshold, recomputes Ithaca distance (Euclidean), updates informational voltage, triggers `initiateIthacaReintegration` if `drift > 0.15 ∨ psi > 0.8`, commits to `ithacaMemory` if `resonance > 0.2`, persists to `localStorage`.
- **Ithaca Reintegration** (lines 343-388): literal implementation of the spec's 5 steps. Correction vector = `qNormalize(0.6·Ithaca ⊕ 0.4·current)`. Psi dampened to ×0.5. Three recursive normalization passes with dot-dependent blend `blend = 0.3 + 0.2·qDot(vector, Ithaca)`.
- **LLM prompt synthesis** (lines 393-418): `buildRHUMContext()` emits a 6-line briefing ending in a **status-dependent directive string** that is prepended to the system prompt. The directives are the sole mechanism by which RHUM's running state modifies Lumos's voice.

Additional surface: `getSnapshot()` (pure read-only copy for the UI panel), `resetIdentity()` (wipes state and localStorage), `saveState`/`loadState` (synchronous JSON serialization).

## 4. The Three Spec Deviations Worth Calling Out

Three points where implementation deliberately softens the spec. All three are documented here so future contributors understand they are **intentional product decisions**, not forgotten TODOs:

1. **Non-resonant input absorbs rather than throws** ([L266-270](../../../src/engine/RHUMEngine.ts#L266-L270)). Spec SI-02 throws `"Entropic Interference Detected"`. A conversational surface cannot crash on noisy user input; the softening preserves chat flow at the cost of strict spec adherence. The energy cost (0.05 destructive energy) preserves the accounting semantic.
2. **Memory commits at `resonance > 0.2` instead of 1:1 coherence** ([L319](../../../src/engine/RHUMEngine.ts#L319)). Spec Condition Gamma demands 1:1 resonance with Ithaca; in practice no real interaction achieves that, so memory would remain empty. The 0.2 threshold admits data at roughly "partial alignment" quality.
3. **Drift threshold 0.15, not 0.05** ([L60](../../../src/engine/RHUMEngine.ts#L60)). Spec SI-01's code skeleton uses `threshold = 0.05`. The live threshold is 3× that. No documented rationale; treat as tuned-by-observation until evidence suggests otherwise.
