## 4. Deep-Dive — The Largest Drift Surfaces

Three places where the implementation narrows the spec. None are *wrong*; all are deliberate scope cuts or v4.0 hooks.

### 4.1 The Instrument-Panel Physics Problem

Eight constants — `UNIVERSAL_TICK_AS`, `LEECH_LATTICE_DIM`, `E7_OBSERVER_SHELL`, `SQRT_180_CLUTCH`, `OBSERVER_7_5D`, `EXTRA_TILT`, `LOST_2_BINDING`, `DARK_MATTER_FRACTION` — are exported from the engine, imported by the panel, and displayed to the operator, but **never read inside `engineTick`**. They are *instrument readouts* reflecting the physics the engine is operating *within*, not drivers of the sim. This is spec-honest (the spec describes a universe these values inhabit) but creates a **UX honesty tension**: the panel suggests these values are live, when in fact they are constants the simulation does not yet couple to. Either wire them into the tick (§5.1, §5.3, §5.5 below) or annotate them in the panel as "environmental constants".

### 4.2 Three-Phase Fold Logic is Compressed to Scalar Relaxation

Spec §4 defines three sequential sub-steps for manifestation: (1) π/4 phase advance in the complex plane, (2) 90° rotation in manifest coordinates, (3) the zipper effect closing triangles in the z-gap. The engine collapses all three into a single scalar first-order decay: `F -= (F - 0.48) × (1/25) × 0.1`. This is computationally fine and converges to the right fixed point, but discards the spec's **geometric mechanism** — calibration is presented as drift rather than as a quaternion rotation with a zipper closure. For a simulator whose job is to make Type 2 physics *feel* real, this is the single largest fidelity gap.

### 4.3 No Persistence — Asymmetric with RHUM

[RHUMEngine.ts](../../../src/engine/RHUMEngine.ts) persists identity state to `localStorage`, making the AI's voice continuous across sessions. PhaseEngine has **no persistence at all** — every page reload starts at `F = 0.50`, `lattice = 1000`, `geometricLock = false`, etc. This matches a "voyage" framing (each session is one flight) but means calibration effort does not accumulate across sessions. Erydir has confirmed this is acceptable per-session simulation behavior; the asymmetry is documented, not corrected.
